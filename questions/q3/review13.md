## 总体评价

这版已经把 review12 的要求落实到位。**Gate 2 可行性阶段可以视为正式结束**，下一步应进入最终无风燃油优化，而不是继续围绕可行性 Gate 反复审计。

当前最核心的判断是：

[
\boxed{\text{可行性已通过，但最优性尚未开始}}
]

也就是说，现在可以写“存在满足预设数值门槛的无风连续可行轨迹”，但还不能写“最省油轨迹”“最优高度-速度规律”或“最终燃油节省率”。

# 四、review12 的边界写得很准确

这版没有犯最常见的错误：把可行性轨迹说成最优轨迹。

文档明确说明：

[
\min s
]

在 (s=0) 后只是“找到任意可行轨迹”，不能写成最省油轨迹，也不能写成 PMP/KKT 已验证的最优解。

下一阶段目标也写对了，应切换为：

[
\max m_f
]

或等价地：

[
\min(m_0-m_f).
]

同时删除质量松弛 (s)，或固定：

[
s=0.
]



这说明现在的任务边界已经清楚：**从可行性问题转向真正的燃油优化问题。**

# 五、最终无风优化的验收边界也列得比较完整

下一阶段要求至少报告：

* 重积分终端质量、高度、速度和状态误差；
* 连续路径约束违反；
* 燃油恒等式残差；
* (J_{61},J_{121},J_{241}) 目标收敛；
* 多初值一致性；
* 求解器成功状态、约束违反、最优性/KKT 信息；
* 无时间约束结果与 (t_f\le1.05/1.10,t_{\mathrm{base}}) 对照；
* 最小推力和近零推力区间比例。

这些都是正确的。尤其是最后两项很关键，因为第三问这个模型有两个天然风险：

1. (t_f) 自由，可能出现“飞得很慢所以省油”的解；
2. (T_{\min}=0)，可能出现不现实的零推力零燃油段。

如果不处理这两个问题，最终燃油优化结果即使数值上更省油，也很难写得可信。

[

]

也就是：

[
\max m_f
]

或：

[
\min(m_0-m_f).
]

推荐还是按：

[
N=61\rightarrow121\rightarrow241
]

continuation，而不是直接从 (N=241) 盲跑。

## 还不能进入

暂时不建议直接进入有风 continuation。

合理顺序是：

1. 无风燃油最优；
2. 无风最优解验证；
3. 无风敏感性；
4. 再用无风最优解 continuation 到有风；
5. 最后比较有风结构变化。
