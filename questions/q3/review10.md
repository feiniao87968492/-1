## 总体评价

这版已经把上一轮的诊断口径基本修正到位。当前结论可以准确概括为：

[
\boxed{\text{离散配点可行，但连续重积分尚未通过}}
]

它已经不是 readiness 或 dry-run，而是正式 Gate 2 的第一轮数值结果。只是目前还不能称为严格可行轨迹，更不能进入最终燃油优化。

# 一、当前失败原因已经定位准确

(N=31,\ h_{\max}=12000\ \mathrm m) 的结果为：

[
s=2.48\times10^{-12}\ \mathrm{kg},
]

[
|\delta_{\mathrm{trap}}|_\infty
=1.42\times10^{-14},
]

离散约束违反为 0。

独立重积分后：

[
m_f^{\mathrm{reint}}
====================

62000.703760\ \mathrm{kg},
]

[
\Delta m_f=+0.703760\ \mathrm{kg},
]

[
\Delta h_f=-0.000994\ \mathrm m,
]

[
\Delta V_f=+0.030218\ \mathrm{m/s}.
]



因此：

* 终端质量实际上仍满足 (m_f\ge62000)；
* 终端高度误差已经远小于 (0.1\ \mathrm m)；
* 唯一明确不通过的是终端速度重积分误差。

速度门槛为：

[
10^{-3}\ \mathrm{m/s},
]

而当前误差是：

[
0.030218\ \mathrm{m/s},
]

约为门槛的：

[
30.2
]

倍。

所以状态改成：

```text
discrete_feasible_reintegration_failed
```

比原来的 `needs_relaxation` 准确得多。这里不需要放宽质量约束，需要提高离散轨迹与连续动力学的一致性。

# 二、重积分诊断已经基本规范

本轮明确了：

```text
control_reconstruction = piecewise_linear_node_controls
```

并增加了：

* 实际重积分终端质量；
* 有符号质量误差；
* 重积分质量短缺；

这是重要改进。此前只报告绝对质量误差 (0.704\ \mathrm{kg})，容易让人误以为连续轨迹违反了质量下限。现在可以看出，连续重积分终端质量反而比离散节点值高约 (0.704\ \mathrm{kg})。

分段线性节点控制也适合当前梯形转录，不需要为了重积分单独引入高阶样条。

# 五、若误差不按网格下降，优先检查这四项

## 1. 重积分模型是否与 NLP 完全一致

必须确认二者使用相同的：

* C1 大气；
* 无风设定 (W=0)；
* 阻力函数；
* 速度惩罚项；
* 推力和航迹角单位；
* 航程域动力学；
* 参数精度。

## 2. ODE 求解器容差是否足够小

应做一次容差敏感性：

[
\texttt{rtol}\in
{10^{-8},10^{-10},10^{-12}}.
]

若终端速度误差随 ODE 容差明显变化，那么测到的不是配点误差，而是重积分器误差。

## 3. 控制是否有节点振荡

建议增加：

[
\max_i|T_{i+1}-T_i|,
\qquad
\max_i|\gamma_{i+1}-\gamma_i|,
]

[
TV(T)=\sum_i|T_{i+1}-T_i|,
\qquad
TV(\gamma)=\sum_i|\gamma_{i+1}-\gamma_i|.
]

Stage 1 只最小化 (s)，在 (s=0) 后目标退化，可能返回控制不够平滑的任意可行解。

## 4. 误差是否集中在最后几个区间

输出每个节点的：

[
V_{\mathrm{reint}}(x_i)-V_i.
]

若误差主要在终点前一两个区间突然增大，应对终端段局部加密，而不必均匀增加全部节点。

# 六、可以增加 Stage 1B，但不应取代网格收敛

Stage 1 已经实现：

[

在得到 (s\approx0) 后，可以加入 Stage 1B：

[
\min R(T,\gamma),
]

其中：

[
R=
\sum_i
\left(
\frac{T_{i+1}-T_i}{T_s}
\right)^2
+
w_\gamma
\sum_i
\left(
\frac{\gamma_{i+1}-\gamma_i}{\gamma_s}
\right)^2.
]

同时必须固定：

[
s=0,
\qquad
m_f\ge62000-\epsilon_{\mathrm{num}}.
]

Stage 1B 可能显著改善分段线性控制的重积分表现，但它不能代替 (N=61/121) 的网格收敛验证。

# 七、方案文档仍有一个内部矛盾

`approach.md` 前面仍写道，第二阶段采用：

[
s\le s_{\min}+\epsilon_s.
]

同时保留：

[
\epsilon_s=10^{-3}\ \mathrm{kg}.
]

但后面又正确规定：

> 若第一阶段达到 (s_{\min}=0)，第二阶段必须固定 (s=0)，不能重新放宽质量约束。

这两个表述存在冲突。

建议将前面的通用表述直接改为：

[
\begin{cases}
s=0, & s_{\min}\le\epsilon_{\mathrm{zero}},[2mm]
s\le s_{\min}+\epsilon_s, & s_{\min}>\epsilon_{\mathrm{zero}}.
\end{cases}
]

对于当前结果，应采用第一种：

[
s=0.
]
