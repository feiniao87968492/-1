## 总体评价

**这版已经具备进入“完整无风可行性配点求解”的条件。**

相比上一版，关键闭环已经建立：

* 质量下限改成终端不等式；
* 数值实现改成航程域；
* 控制变化率与 PMP 口径统一；
* 无风和有风时间基准分开；
* 增加了真正的无风可行性 Gate；
* 没有把未达到 (s=0) 的结果误写成可行或最优。 

当前可以评价为：

> **建模设计基本完成，低维可行性搜索取得接近可行的结果；下一步应升级为完整 collocation 可行性 NLP，而不是继续修改理论框架。**

# 一、Gate 1 的结果是明显进展

无风固定路径的终端质量为：

[
m_f=61163.474\ \mathrm{kg},
]

对应质量缺口：

[
62000-61163.474=836.526\ \mathrm{kg}.
]

低维可行性 Gate 得到：

[
m_f=61989.786\ \mathrm{kg},
\qquad
s^\star=10.214\ \mathrm{kg}.
]



也就是说，参数化轨迹已经消除了约：

[
\frac{836.526-10.214}{836.526}
\approx98.8%
]

的原始质量缺口。

剩余 (10.214\ \mathrm{kg}) 仅相当于题设总燃油 (10450\ \mathrm{kg}) 的约：

[
0.098%.
]

这说明两种可能性：

1. 原问题其实可行，只是当前低维参数化自由度不足；
2. 当前边界下问题确实略微不可行，最优终点质量就在 (62000\ \mathrm{kg}) 附近以下。

现在还不能判断是哪一种。

[
m_f\ge62000\ \mathrm{kg},

[
m_f=61989.786\ \mathrm{kg}.
]

无论缺口多小，严格意义上仍然不可行。

文档已经正确说明：

> Gate 1 不是全局可行性或不可行性证明。

这一表述应保留。

# 三、“最大约束违反为 0”需要改名

结果表目前同时报告：

* 质量松弛 (10.214\ \mathrm{kg})；
* 最大约束违反 (0.000)。



对读者来说这存在表面矛盾：既然终端质量低于 (62000)，怎么会没有约束违反？

实际含义应该是：

> 在引入松弛变量 (s) 后，增广可行性问题的约束全部满足。

因此建议把字段改成：

```text
max_nonrelaxed_constraint_violation
```

或中文：

```text
除终端质量松弛外的最大约束违反
```

同时单独保留：

```text
terminal_mass_shortfall_kg = 10.214
```

这样不会让人误以为原始无松弛问题已经可行。

# 四、(6.94\times10^{-18}) 不是完整配点缺陷证明

当前报告：

[
\text{最大路径残差}=6.94\times10^{-18}.
]



对于“低维参数化控制＋数值积分状态”的实现，这个值更接近：

* ODE 积分内部一致性残差；
* 或由代数重构得到的路径关系残差。

它不能替代未来完整 collocation 中的动力学缺陷：

[
z_{i+1}-z_i-
\int_{x_i}^{x_{i+1}}f(z,u),dx.
]

建议当前命名为：

```text
integration_consistency_residual
```

正式 collocation 完成后再单独报告：

```text
collocation_defect_max
```

两者不要混在一起。

# 五、下一步应使用完整 collocation 可行性 NLP

当前低维参数化已经完成了它的作用：提供一个距离严格可行仅差 (10.214\ \mathrm{kg}) 的良好初值。

下一阶段应把每个节点的状态和控制都作为优化变量：

[
{h_i,V_i,m_i,t_i,T_i,\gamma_i}_{i=0}^{N-1},
]

以及松弛量：

[
s\ge0.
]

目标：

[
\boxed{\min s}
]

约束：

[
m_N+s\ge62000,
]

并满足航程域动力学：

[
\frac{dh}{dx}
=============

\frac{V\sin\gamma}{V_g},
]

[
\frac{dV}{dx}
=============

\frac{(T-D)/m-g\sin\gamma}{V_g},
]

[
\frac{dm}{dx}
=============

-\frac{c_TT\Phi(V)}{V_g},
]

[
\frac{dt}{dx}
=============

\frac1{V_g}.
]

当前航程域转换已经正确写入文档。

# 六、可行性目标达到 (s=0) 后会退化

目标：

[
\min s
]

在达到：

[
s=0
]

以后，所有严格可行轨迹的目标值都相同。

所以完整求解应分为两个阶段：

## 第一阶段：寻找可行轨迹

[
\min s.
]

## 第二阶段：在可行域内最小化油耗

以第一阶段的可行轨迹为初值，求：

[
\max m_f
]

或：

[
\min m_0-m_f.
]

不能在 (s=0) 后继续把最小松弛问题当成正式燃油最优问题。

# 七、必须报告 Gate 1 的飞行包线和活跃约束

现在只报告了：

* 终端质量；
* 松弛量；
* 时间；
* 残差。

但尚未报告这条“接近可行”轨迹是否利用了极端边界。

至少应增加：

| 指标                      | 作用          |
| ----------------------- | ----------- |
| (\min h,\max h)         | 检查高度边界      |
| (\min V,\max V)         | 检查速度边界      |
| (\min T,\max T)         | 检查零推力和推力上限  |
| (\min\gamma,\max\gamma) | 检查航迹角边界     |
| (\max M)                | 检查马赫约束      |
| (\min C_L,\max C_L)     | 检查升力状态      |
| 各约束最小余量                 | 判断哪个约束限制可行性 |

尤其需要知道 (10.214\ \mathrm{kg}) 的缺口是否由以下某个边界造成：

* (V_{\min})；
* (h_{\max})；
* (T_{\min})；
* (T_{\max})；
* (\gamma_{\max})；
* 时间上限。

如果某个边界长期激活，后续应优先对该边界做敏感性，而不是盲目增加节点。

# 八、数值边界仍然是最大未完成项

审计文件仍将“飞行包线冻结”标为 `partial`。

目前材料没有展示最终采用的：

[
h_{\min},h_{\max},
\quad
V_{\min},V_{\max},
]

[
M_{\max},
\quad
T_{\min},T_{\max},
\quad
\gamma_{\max}.

因此还无法判断 Gate 1 的物理合理性。

例如：

* 若速度长期贴着 (V_{\min})，结果高度依赖人为速度下界；
* 若高度贴着 (h_{\max})，结果可能由高度边界决定；
* 若推力接近零，则仍受零推力零油耗缺陷影响。

下一版应在 `results.md` 中直接列出配置值，而不能只说“来自 `configs/default.yaml`”。

\min_x T(x).

[

[

\in

的敏感性测试，才能继续解释结果。
