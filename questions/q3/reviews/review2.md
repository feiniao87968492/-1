# 一、无风预检查非常有价值，但不能据此判定优化问题不可行

预检查得到：

| 场景       |         终点质量 |           油耗 | 是否满足 (m\ge62000) |
| -------- | -----------: | -----------: | ---------------- |
| 配置风场固定路径 | 62022.744 kg | 10427.256 kg | 是                |
| 无风固定路径   | 61163.474 kg | 11286.526 kg | 否                |



这证明了：

[
\text{q1 固定路径在无风下不可行}
]

但不能证明：

[
\text{所有无风轨迹都不可行}.
]

证据文件对这一点说明得很准确：当前检查只验证固定 q1 路径，不证明整个无风优化问题不可行。

无风优化需要相对固定路径减少：

[
11286.526-10450=836.526\ \mathrm{kg},
]

即约减少：

[
7.41%.
]

这个幅度并非显然不可能，但也不能直接假定优化一定能做到。

## 下一步不应立即缩短航程

建议先建立**可行性 NLP**：

[
\min s
]

满足全部动力学、端点和飞行包线约束，并引入松弛变量：

[
m(t_f)+s\ge62000,
\qquad
s\ge0.
]

结果分两种：

* 若 (s^\star=0)，说明原无风问题存在可行轨迹；
* 若 (s^\star>0)，说明在当前边界条件下确实不可行，才需要缩短航程或修改质量约束。

也可以暂时去掉 (m_f\ge62000)，直接最大化 (m_f)。若所得全局或多初值最优结果仍低于 (62000)，则原问题不可行的证据会更强。

# 二、(m\ge62000) 不必作为全程路径约束

当前写成：

[
m(t)\ge62000.
]

由于：

[
\dot m=-c_TT\Phi(V),
]

在：

[
T\ge0,\qquad \Phi(V)>0
]

时，质量必然单调不增。

因此只需要约束：

[
\boxed{m(t_f)\ge62000}
]

即可自动保证全过程：

[
m(t)\ge m(t_f)\ge62000.
]

使用终端不等式而不是路径约束有三个优点：

* 减少每个配点节点上的不等式约束；
* 简化状态约束乘子和 PMP 诊断；
* 避免求解器处理一个实际上只可能在终点激活的路径约束。

除非以后允许负推力或空中加油，否则建议将其改成终端质量约束。

# 三、控制变化率的口径仍有轻微文档冲突

`review.md` 仍写着：

> 直接法中加入 (|dT/dt|)、(|d\gamma/dt|) 约束。

但 `approach.md` 和 `derivation.md` 已经正确说明：

* 若严格加入变化率约束，需要把 (T,\gamma) 扩展为状态；
* 若保留当前四状态 PMP，第一版只能使用控制边界和数值平滑正则。

正式实现前必须选定一个主版本。

## 推荐第一版

状态：

[
z=(x,h,V,m),
]

控制：

[
u=(T,\gamma).
]

只加入：

[
T_{\min}\le T\le T_{\max},
\qquad
|\gamma|\le\gamma_{\max},
]

控制采用分段线性插值，再加入很小的正则项：

[
J_\varepsilon
=============

m_0-m_f+
\varepsilon_T\sum_i(T_{i+1}-T_i)^2+
\varepsilon_\gamma\sum_i(\gamma_{i+1}-\gamma_i)^2.
]

然后对：

[
\varepsilon_T,\varepsilon_\gamma\to0
]

进行敏感性检查，确认最优油耗不是由正则项决定的。

严格变化率约束留到扩展版本。这样当前 PMP 推导可直接使用。

# 四、建议将直接求解改为航程域

当前直接法仍采用时间归一化：

[
\tau=t/t_f.
]

这个方案正确，但本问题的终端航程固定，使用 (x) 作为独立变量会更自然。

只要：

[
V_g=V\cos\gamma+W(h)>0,
]

就有：

[
\frac{dh}{dx}
=============

\frac{V\sin\gamma}{V_g},
]

[
\frac{dV}{dx}
=============

\frac{(T-D)/m-g\sin\gamma}{V_g},
]

[
\frac{dm}{dx}
=============

-\frac{c_TT\Phi(V)}{V_g},
]

并增加飞行时间状态：

[
\frac{dt}{dx}
=============

\frac1{V_g}.
]

于是：

[
x\in[0,X_f]
]

是固定计算区间，不再需要：

* 把 (x) 作为状态；
* 把 (t_f) 作为优化变量；
* 单独施加终端航程等式。

这样状态减少为：

[
(h,V,m,t),
]

控制仍是：

[
(T,\gamma).
]

运营时间约束可以直接写成：

[
t(X_f)\le1.05t_{\mathrm{base}}.
]

建议：

* PMP 理论推导继续保留时间域；
* 直接配点数值实现改用航程域；
* 最后验证时间域和航程域动力学一致性。

# 六、零推力问题仍只是“识别”，尚未解决

当前已经准确指出：

[
T_{\min}=0
]

会导致：

[
T=0\Rightarrow\dot m=0,
]

从而鼓励无油耗滑翔和推力切换。

但正式模型仍需选择一个主方案。

建议采用：

[
T_{\min}=T_{\mathrm{idle}}>0,
]

其中 (T_{\mathrm{idle}}) 作为基准推力的可配置比例，并进行敏感性分析。

相比增加：

[
q_{\mathrm{idle}}
]

这种新燃油参数，设置正的推力下界对原题燃油公式改动更小。

应至少计算：

[
T_{\mathrm{idle}}
\in
{0,\ 0.05T_{\mathrm{ref}},\ 0.10T_{\mathrm{ref}}}.
]

如果最终解完全不接近推力下界，这一假设影响较小；若长期贴着下界，则必须把它列为主敏感参数。

# 七、PMP 诊断还缺少伴随变量的获取方法

当前计划计算：

* (\partial H/\partial T)；
* (\partial H/\partial\gamma)；
* Hamiltonian 沿程变化；
* (H(t_f))。

但这些量需要伴随变量：

[
\lambda_x,\lambda_h,\lambda_V,\lambda_m.
]

仅从状态和控制轨迹无法计算。

正式实现必须说明伴随变量来源：

1. 从 IPOPT/CasADi 的动力学缺陷约束乘子提取离散伴随量；
2. 或固定最优状态—控制轨迹后，反向积分连续伴随方程；
3. 或求解离散 KKT 系统。

推荐优先使用 NLP 约束乘子，并注意尺度化后需把乘子恢复到物理尺度。

此外，Hamiltonian 不应简单要求全程完全平坦：

* 在无显式时间、无活动路径约束的光滑弧段，应近似常数；
* 控制切换点、状态约束进入/退出点可能出现分段结构；
* 自由终端时间且未触及时间边界时才要求 (H(t_f)\approx0)。

# 八、自由时间对照中的 (t_{\mathrm{base}}) 必须按场景区分

当前预检查已经得到：

[
t_{\mathrm{base,wind}}=721.753\ \mathrm s,
]

[
t_{\mathrm{base,no\ wind}}=790.755\ \mathrm s.
]



因此时间约束应分别写成：

### 无风问题

[
t_f\le1.05\times790.755.
]

### 有风问题

[
t_f\le1.05\times721.753.
]

不能统一使用 q2 有风时间作为无风运营基准，否则会把顺风收益误当成优化器必须满足的飞行时间要求。
