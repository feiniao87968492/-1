## 总体评价

**这是一次实质性进展：正式非 dry-run 可行性 NLP 已经真正跑通。**

当前结果证明了两件事：

1. 航程域梯形配点 NLP 能找到离散意义下满足终端质量约束的轨迹；
2. 当前 (N=31) 网格还不足以证明该轨迹在连续动力学下严格可行。

因此，项目已经从“求解器准备阶段”进入：

[
\boxed{\text{正式求解中的网格收敛与连续一致性验证阶段}}
]

但 Gate 2 仍然没有通过。文档对此没有夸大：虽然质量松弛和配点缺陷都接近零，但重积分误差未达门槛，所以仍未进入最终无风燃油优化。

# 一、离散 NLP 本身求解得很好

(N=31) 的结果是：

[
s=2.48\times10^{-12}\ \mathrm{kg},
]

[
|\delta_{\mathrm{trap}}|_\infty
=1.42\times10^{-14},
]

[
\text{最大尺度化约束违反}=0.
]



这说明：

* SLSQP 成功找到了一个离散可行点；
* 终端质量约束在节点模型中满足；
* 梯形配点等式被压到了机器精度附近；
* 变量尺度和约束转录至少在 (N=31) 下是可工作的。

从此前低维 Gate 的：

[
s=10.214\ \mathrm{kg}
]

下降到离散 NLP 的数值零，说明增加完整节点自由度确实有效。

# 二、当前失败不是“质量不可行”，而是“离散化精度不足”

独立重积分得到：

[
|\Delta m_f|=0.703760\ \mathrm{kg},
]

[
|\Delta h_f|=0.000994\ \mathrm m,
]

[
|\Delta V_f|=0.030218\ \mathrm{m/s}.
]



其中：

* 高度误差已经很小；
* 质量误差还需下降；
* 速度误差相对于 (10^{-3}\ \mathrm{m/s}) 门槛约大 (30) 倍。

所以当前问题不是必须“放宽质量约束”，而是：

[
\boxed{\text{离散轨迹和连续 ODE 轨迹尚未充分一致}}
]

建议把当前状态从：

```text
needs_relaxation
```

改为更准确的：

```text
needs_mesh_refinement
```

或：

```text
discrete_feasible_reintegration_failed
```

因为当前质量松弛已经是零，不需要 relaxation；需要的是网格加密、控制重构或更高阶转录。

# 三、(N=31) 的网格本来就很粗

航程为：

[
X_f=189781.310\ \mathrm m.
]

(N=31) 只有 30 个区间，因此每个区间长度约为：

[
\Delta x
========

\frac{189781.310}{30}
\approx6326\ \mathrm m.
]

以约 (240\ \mathrm{m/s}) 飞行，相当于每个区间约覆盖：

[
26\ \mathrm s.
]

对于同时变化的：

[
h,\quad V,\quad m,\quad T,\quad\gamma,
]

这个网格相当粗。出现 (0.0302\ \mathrm{m/s}) 的重积分终端速度误差并不异常。

因此，这一轮结果更接近：

> **证明转录和求解器链路正确，而不是证明最终连续轨迹已可行。**

# 四、网格加密后的误差可以先做一个粗略预判

梯形法通常具有二阶全局精度。若当前误差主要来自正常的梯形离散误差，每次将区间数量加倍，误差应大约下降到原来的四分之一。

按此粗略估计：

| 网格      | 质量重积分误差估计 |   速度重积分误差估计 |
| ------- | --------: | ----------: |
| (N=31)  |  0.704 kg |  0.0302 m/s |
| (N=61)  |  0.176 kg | 0.00755 m/s |
| (N=121) |  0.044 kg | 0.00189 m/s |
| (N=241) |  0.011 kg | 0.00047 m/s |

这只是二阶收敛假设下的推算，但它提示：

* (N=61) 大概率仍不够；
* (N=121) 的质量误差可能已经很好；
* (N=121) 的速度误差仍可能略高于 (10^{-3}\ \mathrm{m/s})；
* 最终可能需要 (N=241)，或者改用 Hermite–Simpson。

因此不要在 (N=61) 仍未通过时认为模型有问题。

# 五、下一步首先应验证收敛阶，而不是直接换算法

依次运行：

[
N=61,\qquad N=121.
]

计算误差比：

[
r_{31\to61}
===========

\frac{e_{31}}{e_{61}},
\qquad
r_{61\to121}
============

\frac{e_{61}}{e_{121}}.
]

若：

[
r\approx4,
]

说明梯形法表现为正常二阶收敛，应继续加密。

若误差基本不降，或下降十分不规则，则应检查：

* 重积分时使用的控制插值方式；
* NLP 中的控制是否出现节点振荡；
* 配点模型与重积分模型是否使用同一个大气和阻力函数；
* 重积分是否使用了相同的 (T(x),\gamma(x)) 重构；
* 状态尺度恢复是否一致。

不能只看求解器报告的 (10^{-14}) 配点残差。

# 六、必须明确重积分时如何重构控制

当前状态和控制只定义在节点上。独立 ODE 重积分需要连续的：

[
T(x),\qquad\gamma(x).
]

正式文档应明确采用：

* 分段常数；
* 分段线性；
* 三次插值；
* 或其他重构。

建议第一版使用与梯形法最自然匹配的**分段线性控制**。

同时要避免高阶普通三次样条产生：

* 推力超出上下界；
* 航迹角超出边界；
* 节点之间振荡。

重积分不仅要检查终端误差，还应在密集网格检查：

[
T_{\min}\le T(x)\le T_{\max},
]

[
|\gamma(x)|\le\gamma_{\max},
]

[
h(x)\le h_{\max},
\qquad
M(x)\le M_{\max}.
]

# 七、第一阶段目标为 (\min s) 会产生大量退化解

一旦达到：

[
s=0,
]

目标函数对所有离散可行轨迹都相同。

因此 SLSQP 可能返回一条：

* 离散约束完全满足；
* 但控制变化较剧烈；
* 连续重积分表现不够好的轨迹。

这不代表 SLSQP 求错了，而是第一阶段目标本身不选择“最平滑、最容易连续重构”的解。

## 建议增加 Gate 2 的阶段 1B

在保持硬约束：

[
s=0,\qquad m_f\ge62000
]

的条件下，最小化：

[
R=
w_T\sum_i
\left(
\frac{T_{i+1}-T_i}{T_s}
\right)^2
+
w_\gamma\sum_i
\left(
\frac{\gamma_{i+1}-\gamma_i}{\gamma_s}
\right)^2.
]

这不是最终燃油优化，而是在离散可行解集合中选择更平滑的代表。

顺序应为：

1. Stage 1A：(\min s)；
2. Stage 1B：固定 (s=0)，最小化控制平滑；
3. 独立重积分；
4. 网格加密；
5. 通过后才进入正式燃油目标。

当前方案已经明确第二阶段不能重新放开质量松弛，这一原则是正确的。

# 八、重积分质量误差必须同时报告符号和实际终端质量

现在报告的是：

```text
reintegration_terminal_mass_error_kg = 0.703760
```

看起来是绝对值。

但连续可行性真正关心的是：

[
m_{f,\mathrm{reint}}\ge62000.
]

若重积分结果为：

[
m_{f,\mathrm{reint}}=61999.296,
]

则连续轨迹违反质量下限。

若为：

[
m_{f,\mathrm{reint}}=62000.704,
]

则质量约束其实仍满足，只是与配点状态不一致。

建议新增：

```text
reintegration_terminal_mass_kg
reintegration_terminal_mass_signed_error_kg
reintegration_terminal_mass_shortfall_kg
```

速度和高度也应同时报告有符号误差，以判断偏差方向。

# 十一、SLSQP 可以继续用于 (N=61)，但要准备备用求解器

当前 (N=31) 约有：

[
4N+2N+1=187
]

个主要优化变量，SLSQP 可以处理。

到 (N=121) 时变量数约为：

[
727.
]

再叠加大量等式和不等式约束后，SLSQP 可能出现：

* 计算时间明显增加；
* 数值差分梯度误差；
* 约束雅可比病态；
* 迭代停滞；
* 难以稳定提取乘子。

建议：

* (N=61) 先继续使用 SLSQP；
* (N=121) 若明显不稳定，优先迁移到 CasADi/IPOPT 或其他支持稀疏导数的 NLP 求解器；
* 不要因为 SLSQP 失败就修改物理模型或放宽约束。
