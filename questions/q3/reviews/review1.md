## 总体评价

**第三问第一轮设计总体正确，可以进入求解器实现前的修订阶段。**

目前完成得较好的部分包括：

* 正确识别固定终止质量会使目标函数退化；
* 固定终端高度和速度，避免通过降低终端机械能获得虚假省油；
* 将 (h,V) 作为状态，将 (T,\gamma) 作为操作量；
* Hamiltonian、伴随方程和阻力偏导的代数结构基本正确；
* 采用直接配点法主求解、PMP 辅助诊断的路线合理；
* 当前状态标记为 `in_design`，没有提前伪造最优数值。 

但现在**还不适合直接开始正式求解**。至少有四个必须先解决的问题，其中第一个可能直接导致无风优化不可行。

# 一、最关键问题：q2 航程未必适合无风问题

当前第三问固定：

[
X_f=189781.310\ \mathrm m,
]

并要求：

[
m(t)\ge62000\ \mathrm{kg}.
]

同时把 q2 基线

[
J=10427.256\ \mathrm{kg},\qquad m_f=62022.744\ \mathrm{kg}
]

作为可行初值和比较基线。

但 q2 基线使用了顺风场：

[
W(h)=20+3\times10^{-5}(h-10000)^2.
]

第三问却首先求解：

[
W(h)=0
]

的无风最优问题。相同路径在无风下：

* 地速下降；
* 完成相同地面航程所需时间增加；
* 总油耗通常增加；
* q2 的质量剖面不再满足无风动力学。

因此：

> **q2 有风基线不是无风优化问题的可行基线。**

当前文档中“基线必须是优化问题的可行解”的要求是正确的，但尚未真正实现。

## 必须先做的预检查

沿固定参考路径：

[
h=h_{\mathrm{ref}}(x),\qquad V=240\ \mathrm{m/s},
]

重新设置：

[
W=0,
]

并重新积分质量和时间，得到：

[
J_{\mathrm{base,no\ wind}},
\qquad
m_{\mathrm{base,no\ wind}}(X_f).
]

然后判断：

[
m_{\mathrm{base,no\ wind}}(X_f)\ge62000
]

是否成立。

### 若不成立，有三种选择

1. 缩短第三问航程，使无风基线也可行；
2. 保留航程，但取消 (m\ge62000) 这一硬约束，仅最大化终端质量；
3. 证明最优轨迹可以在该航程下保持 (m_f\ge62000)，但此时无法使用基线作为可行初值。

最稳妥的是重新定义：

[
X_{f,3}
=======

\min
\left(
X_{\mathrm{robust,q2}},
X_{\mathrm{base,no\ wind}}
\right).
]

# 二、控制变化率约束与 PMP 推导不完全一致

当前直接法准备加入：

[
|\dot T|\le r_T,
\qquad
|\dot\gamma|\le r_\gamma,
]

但 PMP 推导仍然把 (T,\gamma) 当作普通控制量，并使用：

[
\frac{\partial H}{\partial T}=0,
\qquad
\frac{\partial H}{\partial\gamma}=0.
]



当控制变化率约束激活时，这两个驻值条件不能直接作为完整诊断条件。

## 严格处理方式

将推力和航迹角扩展为状态：

[
z_T=T,\qquad z_\gamma=\gamma,
]

新增控制：

[
u_T=\dot T,\qquad u_\gamma=\dot\gamma.
]

扩展状态方程：

[
\dot T=u_T,
\qquad
\dot\gamma=u_\gamma,
]

并约束：

[
|u_T|\le r_T,
\qquad
|u_\gamma|\le r_\gamma.
]

此时优化状态成为：

[
(x,h,V,m,T,\gamma),
]

真正控制量成为：

[
(u_T,u_\gamma).
]

PMP 应对 (u_T,u_\gamma) 写极小条件，而 (T,\gamma) 会拥有各自的伴随方程。

---

## 简化处理方式

第一版先不加入连续控制变化率约束，只使用：

* 控制上下界；
* 分段线性控制；
* 网格加密；
* 很小的控制平滑正则项。

此时当前 PMP 推导基本可以保留。

但不能一方面加入严格的 (\dot T,\dot\gamma) 约束，另一方面仍用简单的：

[
\partial H/\partial T,\quad
\partial H/\partial\gamma
]

作为全部 PMP 验证。

# 三、自由终端时间可能产生不符合运营意义的最优解

当前设置：

[
t_f\ \text{自由},
]

目标仅为：

[
\min J=m_0-m(t_f).
]



这在数学上成立，但优化器可能选择：

* 接近最低允许速度；
* 更长的飞行时间；
* 较低推力；
* 以牺牲时间换取燃油节省。

这得到的是纯粹的**最小燃油轨迹**，不一定是航空公司的合理巡航策略。

## 建议采用两层结果

### 主结果

保持自由终端时间，求理论最低油耗：

[
\min J.
]

### 运营约束对照

加入：

[
t_f\le(1+\delta_t)t_{\mathrm{base}},
]

例如：

[
\delta_t=5%,10%.
]

或者采用加权目标：

[
J_\alpha
========

m_0-m(t_f)+\alpha t_f.
]

这样可以回答：

> 燃油节省是否只是依靠大幅延长飞行时间获得的？

至少应将总飞行时间作为核心结果报告。

# 四、线性推力油耗模型会鼓励非物理控制结构

当前油耗率为：

[
q_f
===

c_TT\Phi(V).
]

如果允许：

[
T_{\min}=0,
]

则：

[
T=0\Rightarrow q_f=0.
]

这可能促使优化器形成：

* 低推力或零推力滑行；
* 随后高推力补充机械能；
* 推力边界间切换的 bang-bang 结构。

当前文档也已经意识到推力可能落在边界。

但仅靠变化率约束不能解决燃油模型本身缺少慢车油耗的问题。

## 建议至少采用一种处理

### 方法 A：设置正的慢车推力下限

[
T\ge T_{\mathrm{idle}}>0.
]

继续使用题面燃油模型。

### 方法 B：加入基础燃油流量

[
q_f
===

q_{\mathrm{idle}}
+
c_TT\Phi(V).
]

### 方法 C：保留原模型但进行解释

若必须严格采用题面公式，应明确：

> 模型忽略发动机慢车基础油耗，推力边界弧属于题面简化模型下的数学结果，不应直接解释为真实发动机最优操作。

同时对 (T_{\min}) 做敏感性分析。

# 五、风切变目前只按“运动学风场”处理

当前有风模型是：

[
\dot x=V\cos\gamma+W(h),
]

而：

[
\dot V
======

\frac{T-D}{m}-g\sin\gamma
]

没有 (W'(h)) 项。

这属于：

> **运动学风场模型：风只改变地速和单位航程时间，不直接改变空速动力学。**

作为竞赛基线是可以接受的，但题目专门要求分析“垂直风切变出现在哪些方程中”，当前回答只覆盖了：

* (W(h)) 进入 (\dot x)；
* (W'(h)) 进入 (\dot\lambda_h)。



更高精度的真空速动力学中，水平风随高度变化还会产生风梯度加速度。沿空速方向投影可得到近似项：

[
\dot V
======

\frac{T-D}{m}
-g\sin\gamma
-W'(h)V\sin\gamma\cos\gamma.
]

因此建议在文档中区分：

### 主模型

运动学风场：

[
W(h)\text{ 只进入 }\dot x.
]

### 扩展模型

动力学风切变：

[
W'(h)\text{ 进入空速和航迹角动力学}.
]

如果继续将 (\gamma) 直接作为控制量，主模型应明确定位为小角度准稳态简化。更完整的风切变模型通常应把 (\gamma) 作为状态，把 (C_L) 或迎角作为控制。

# 六、风场会强烈推动高度边界解

风场为：

[
W(h)
====

20+3\times10^{-5}(h-10000)^2.
]

其最小值在 (h=10000\ \mathrm m)。

例如：

[
W(9000)=W(11000)=50\ \mathrm{m/s},
]

[
W(8000)=W(12000)=140\ \mathrm{m/s}.
]

如果高度边界设置为：

[
8000\le h\le12000\ \mathrm m,
]

优化器可能主动向上下边界移动，以获取极高顺风，而不是因为气动效率选择高度。

因此有风结果很可能是：

> **由风场边界驱动的最优解，而不是由飞机气动效率驱动的内部最优解。**

这不一定是计算错误，但必须：

1. 先绘制可行高度区间内的 (W(h))；
2. 报告高度约束是否激活；
3. 对高度边界做敏感性；
4. 比较无风和有风解的活跃约束；
5. 不把贴边轨迹简单解释为普遍航空规律。

# 七、直接法必须增加变量尺度化

目前变量数量级差异很大：

[
x\sim10^5,
\quad
h\sim10^4,
\quad
m\sim10^4,
\quad
T\sim10^4,
\quad
V\sim10^2,
\quad
\gamma\sim10^{-2}.
]

当前方案只说明全部使用 SI 单位。

直接把这些变量送入 NLP 求解器容易导致：

* 雅可比病态；
* 约束容差失衡；
* 求解器误判收敛；
* Hamiltonian 诊断受数值尺度影响。

必须定义无量纲变量，例如：

[
\bar x=\frac{x}{X_f},
\qquad
\bar h=\frac{h}{10^4},
\qquad
\bar V=\frac{V}{240},
]

[
\bar m=\frac{m}{72450},
\qquad
\bar T=\frac{T}{5\times10^4},
\qquad
\bar t=\frac{t}{800}.
]

NLP 内部使用尺度化变量，最终输出再恢复 SI 单位。

# 八、11 km 大气层边界在第三问中需要重新检查

第二问固定路径低于 11 km，但第三问高度是优化变量。

如果设置：

[
h_{\max}>11000\ \mathrm m,
]

优化轨迹可能穿过标准大气层边界。

当前 PMP 推导假设：

[
\rho(h)
]

可平滑求导，并直接使用 (d\rho/dh)。

分层 ISA 在 11 km 处虽然温度和压力连续，但温度梯度发生变化，(\rho'(h)) 的表达也会切换。

第一版建议二选一：

* 暂时限制
  [
  h_{\max}<11000\ \mathrm m;
  ]
* 或对 11 km 附近做平滑过渡，使大气函数至少 (C^1) 连续。

否则梯度型求解器和 PMP 残差可能在层边界附近不稳定。

# 九、能量高度降维的条件需要再收紧

当前写道，无风且目标和约束只依赖机械能时，可以考虑能量高度降维。

方向正确，但还应增加一个关键条件：

> 阻力和发动机油耗不能任意分别依赖 (h,V)，或者必须能在固定能量高度下，通过一个内层优化消去高度—速度分配。

因为：

[
E=h+\frac{V^2}{2g}
]

相同，并不代表：

[
\rho(h),\quad D(h,V),\quad \Phi(V)
]

相同。

因此能量高度本身并不会自动将二维问题变成一维问题。

更准确的表述是：

[
\mathcal L_{\mathrm{red}}(E,\dot E)
===================================

\min_{\substack{h,V\h+V^2/(2g)=E}}
\mathcal L(h,V,\dot E).
]

只有这个内层最小化能够唯一、平滑地确定能量分配时，才可能形成真正的一维降维模型。

# 十、PMP 推导本身的评价

在以下前提下：

* 运动学风场；
* (\gamma) 直接作为控制；
* 不考虑控制变化率约束；
* 大气函数在当前弧段可微；

当前伴随方程和阻力导数总体正确。

特别是：

[
\dot\lambda_h
=============

-\lambda_xW'(h)
+
\frac{\lambda_V}{m}
\frac{\partial D}{\partial h},
]

[
\dot\lambda_m
=============

\lambda_V
\left[
\frac{T-D}{m^2}
+
\frac1m\frac{\partial D}{\partial m}
\right]
]

符号没有明显错误。

自由终端时间下：

[
H(t_f)=0
]

也成立，但应注明这是在：

* 终端约束不显含时间；
* 使用正确的增广 Hamiltonian；
* 不存在终端时间上下界激活；

的条件下成立。
