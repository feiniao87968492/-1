# 一、Gate 1 的证据解释现在是正确的

当前结果为：

[
m_f=61989.786\ \mathrm{kg},
\qquad
s^\star=10.214\ \mathrm{kg},
]

总时间为：

[
t_f=802.883\ \mathrm s.
]

非松弛约束最大违反为 0，积分一致性残差约为：

[
6.94\times10^{-18}.
]



现在的字段解释不再矛盾：

* 原始终端质量约束仍缺少 (10.214\ \mathrm{kg})；
* 引入松弛变量以后，其他约束全部满足；
* 极小残差只说明数值积分与参数化轨迹一致；
* 它不代表完整配点动力学缺陷已经通过。

这一部分已经可以保留。

# 二、当前最重要的新发现：高度上界决定了 Gate 1

Gate 1 的轨迹范围是：

[
9500\le h\le12000\ \mathrm m,
]

并且高度上界余量只有：

[
2.98\times10^{-5}\ \mathrm m.
]

也就是说，优化轨迹实际上贴在：

[
h_{\max}=12000\ \mathrm m
]

上。其他主要约束均有较明显余量：

* 最小速度余量 (24.373\ \mathrm{m/s})；
* 推力上界余量 (17981.830\ \mathrm N)；
* 航迹角余量 (0.014859\ \mathrm{rad})；
* 马赫数余量 (0.031276)。

因此当前 (10.214\ \mathrm{kg}) 的缺口，不应只理解成“低维参数化自由度不足”，还可能是：

> **人为设置的高度上界限制了进一步降低油耗的能力。**

## 这意味着什么

若将：

[
h_{\max}=12000
]

增加到 (12200) 或 (12500\ \mathrm m)，低维 Gate 可能直接达到 (s=0)。

反过来，若为了避免 11 km 大气层导数问题而限制：

[
h_{\max}<11000\ \mathrm m,
]

质量缺口可能显著扩大。

所以 (s^\star=10.214\ \mathrm{kg}) 目前不是对模型可行性的稳健结论，而是对以下配置的结论：

[
h_{\max}=12000\ \mathrm m.
]

# 三、12 km 轨迹与 11 km 大气层处理存在直接冲突

当前方案写道：

> 若 (h_{\max}>11000\ \mathrm m)，需使用可微分层大气或限制 (h_{\max}<11000\ \mathrm m)。

但 Gate 1 已经达到：

[
h=12000\ \mathrm m.
]

因此轨迹必然：

1. 从 9500 m 上升并穿过 11000 m；
2. 在高空段达到 12000 m；
3. 再下降到终端高度 10577.124 m；
4. 第二次穿过 11000 m。

这不是未来才可能发生，而是当前 Gate 1 已经实际发生。

## 完整 collocation 前必须选一种处理

### 方案 A：限制对流层内

设置：

[
h_{\max}=10950\ \mathrm m.
]

优点是模型光滑、PMP 简单；缺点是很可能增加质量缺口。

### 方案 B：采用 (C^1) 平滑层间过渡

在 11 km 附近设置一段平滑过渡，例如：

[
10950\le h\le11050\ \mathrm m.
]

保证：

[
\rho(h),\qquad \frac{d\rho}{dh}
]

连续。

这是最适合自动微分和单阶段配点的方法。

### 方案 C：多阶段配点

把 11 km 以下和以上作为不同 phase，并设置状态连续条件。

物理表达最严格，但当前工程复杂度较高。

**竞赛实现建议采用方案 B。**

# 四、航程域 NLP 的乘子不能直接当成时间域 PMP 伴随变量

当前理论推导使用时间域 Hamiltonian：

[
H_t
===

\lambda_x\dot x+
\lambda_h\dot h+
\lambda_V\dot V+
\lambda_m\dot m.
]

数值实现则使用航程域状态：

[
(h,V,m,t)
]

和动力学：

[
\frac{dh}{dx},\quad
\frac{dV}{dx},\quad
\frac{dm}{dx},\quad
\frac{dt}{dx}.
]

这个转化本身是正确的。

但完整 collocation 的动力学缺陷乘子，对应的是**航程域伴随变量**，不能只做单位尺度恢复后直接代入时间域 Hamiltonian。

航程域 Hamiltonian 应写为：

[
\begin{aligned}
H_x={}&
\mu_h\frac{V\sin\gamma}{V_g}\
&+\mu_V
\frac{(T-D)/m-g\sin\gamma}{V_g}\
&-\mu_m
\frac{c_TT\Phi(V)}{V_g}\
&+\mu_t\frac1{V_g},
\end{aligned}
]

其中：

[
V_g=V\cos\gamma+W(h).
]

控制驻值条件应首先检查：

[
\frac{\partial H_x}{\partial T},
\qquad
\frac{\partial H_x}{\partial\gamma}.
]

特别是 (V_g) 也依赖 (\gamma)，所以航程域的：

[
\frac{\partial H_x}{\partial\gamma}
]

会包含分母导数，不能直接使用时间域的 (\partial H_t/\partial\gamma)。

当前文档只说明从 NLP 约束乘子提取伴随变量并恢复尺度，还缺少这个“独立变量变换”步骤。

## 推荐处理

* PMP 理论章节保留时间域形式；
* collocation 数值诊断首先使用航程域 KKT/Hamiltonian；
* 另行推导时间域与航程域伴随量的映射；
* 映射未完成前，不直接声称时间域 Hamiltonian 平坦性验证通过。

这不影响 Gate 2 的可行性求解，但会影响后续正式最优解验证。

# 五、可行性目标和控制平滑项必须采用分层优化

当前第一版准备使用相邻控制平滑正则。

在可行性 Gate 中不能简单使用：

[
\min\left[
s+
\varepsilon R(T,\gamma)
\right].
]

否则即使存在 (s=0) 的稍微不平滑轨迹，求解器也可能为了降低 (R) 而接受一个正的质量缺口。

## 推荐采用词典序两阶段

### 阶段 1

[
\min s.
]

得到：

[
s_{\min}.
]

### 阶段 2

固定：

[
s\le s_{\min}+\varepsilon_s,
]

再求：

[
\min R(T,\gamma).
]

这样可以保证控制平滑不会牺牲可行性。

也可以先完全不加平滑项，通过网格加密观察控制是否出现节点振荡。

# 六、必须定义“(s^\star\approx0)”的数值标准

目前文档反复写：

[
s^\star\approx0,
]

但没有明确阈值。

建议将 Gate 通过标准固定为：

[
s^\star\le0.05\ \mathrm{kg},
]

同时满足：

[
|\text{collocation defect}|_\infty
\le10^{-6}
]

——这里使用尺度化缺陷——并且：

[
|h(X_f)-h_f|\le0.1\ \mathrm m,
]

[
|V(X_f)-V_f|\le10^{-3}\ \mathrm{m/s}.
]

还要满足：

[
\text{所有状态和控制余量}\ge-\epsilon_{\mathrm{con}}.
]

具体阈值可以根据求解器精度调整，但必须预先写入验证文件，不能在看完结果后临时决定什么叫“约等于零”。
